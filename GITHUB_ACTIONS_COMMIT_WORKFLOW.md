# GitHub Actions 代码评审 - 写入 Commit 版本

## 🎯 工作流程说明

这个版本的 GitHub Actions 工作流会在推送代码时自动触发，调用大模型接口进行代码评审，并将评审结果写入到新的 commit 中。

### 触发条件

- ✅ **Push 事件**: 推送代码到 `main` 或 `develop` 分支
- ❌ **Pull Request 事件**: 不处理 PR
- 🚫 **防止循环**: 自动跳过包含 `[skip ci]` 或 `🤖 AI Code Review:` 的 commit

## 📋 关键特性

### 1. **自动跳过评审 commit**
```yaml
if: "!contains(github.event.head_commit.message, '[skip ci]') &&
     !contains(github.event.head_commit.message, '🤖 AI Code Review:')"
```
- 自动检测并跳过由机器人创建的 commit
- 防止无限循环触发

### 2. **创建独立的评审结果文件**
```bash
cat > AI_REVIEW_RESULT.md << 'EOF'
## 🤖 AI Code Review Report
...
EOF
```
- 评审结果保存在 `AI_REVIEW_RESULT.md` 文件中
- Markdown 格式，便于查看

### 3. **防止循环触发**
```bash
git commit -m "🤖 AI Code Review: ... [skip ci]"
```
- 使用 `[skip ci]` 标记防止 GitHub Actions 重新触发
- 双重保护：commit message 检查 + [skip ci] 标记

## 🚀 使用说明

### 1. 正常推送代码
```bash
# 修改代码
git add .
git commit -m "feat: add new feature"
git push origin main
```

**结果：**
1. GitHub Actions 自动触发
2. 执行代码评审
3. 创建新的 commit 包含评审结果
4. 评审结果推送到远程仓库

### 2. 查看评审结果
```bash
# 拉取最新代码
git pull origin main

# 查看评审结果文件
cat AI_REVIEW_RESULT.md

# 或在 GitHub 上查看
# 打开 AI_REVIEW_RESULT.md 文件
```

### 3. Git 历史查看
```bash
# 查看完整的 commit 历史
git log --oneline

# 输出示例：
# def456 🤖 AI Code Review: abc123def456
# abc123 feat: add new feature
# 789abc fix: bug fix
# 456def 🤖 AI Code Review: 789abc
```

## ⚙️ 配置选项

### 修改触发分支
```yaml
on:
  push:
    branches:
      - main      # 主分支
      - develop   # 开发分支
      # - feature/*  # 功能分支（可选）
```

### 修改评审模型
```yaml
-d '{
  "model": "glm-4-flash"  # 可选: glm-4, glm-4-7-flash
}'
```

### 修改文件名
```yaml
# 在 Commit Review Results 步骤中
cat > MY_REVIEW.md << 'EOF'
```

## 🛠️ 故障排查

### 问题 1: 循环触发
**解决方案：**
- 确保使用 `[skip ci]` 标记
- 检查 `if` 条件是否正确

### 问题 2: 评审结果未提交
**解决方案：**
- 检查 API 调用是否成功
- 查看错误日志
- 验证 Git 配置

### 问题 3: 权限错误
**解决方案：**
- 确保 `permissions: contents: write`
- 检查 GITHUB_TOKEN 权限
